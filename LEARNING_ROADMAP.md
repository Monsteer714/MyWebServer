### 面试常见追问（WebServer 相关）

1. epoll 的水平触发和边缘触发区别？为什么用 EPOLLONESHOT？
2. 线程池怎么保证线程安全？为什么用 pthread 而不是 std::thread？
3. 10000 个连接同时到达，服务器会怎么表现？
4. 定时器 O(n) 插入，连接数大了怎么优化？（时间轮/最小堆）
5. Reactor 和 Proactor 的区别？你的代码属于哪种？
6. 日志系统是异步的，如果日志队列满了怎么办？
7. mmap 和 read/write 的区别？零拷贝是怎么做到的？

## 第二阶段：C++ 和系统知识深度

### C++ 深度

- 对象模型：虚函数表、多重继承内存布局
- C++11/14/17/20：move 语义、智能指针、lambda 原理、concept
- STL 源码：vector 扩容、deque 分段、unordered_map 的 hash 冲突
- 内存管理：tcmalloc/jemalloc 原理、内存池、placement new
- 编译链接：静态/动态链接、符号重定位、PIC/PIE

### 系统深度

- 进程/线程模型、clone、信号、内核调度 CFS
- 虚拟内存 → mmap/madvise/mlock、缺页中断
- 文件系统 → inode、ext4 日志、VFS
- 网络栈 → TCP 拥塞控制、epoll 在内核中的位置
- 锁/同步机制 → 互斥锁、条件变量、无锁队列

### 方向一：基础架构 / 分布式存储 ⭐⭐⭐⭐⭐

**典型岗位：** 分布式存储引擎、数据库内核、消息队列、RPC 框架、容器运行时

**代表公司：** 字节跳动（基础架构组）、阿里云（PolarDB/Flink/Lindorm）、腾讯云（TDSQL/TBase）、PingCAP（TiDB）、OceanBase、华为（高斯 DB）

**你需要的能力：**
- 网络编程 / 多线程 / 锁 / 无锁数据结构 ← 你正在做
- Raft/Paxos 等分布式一致性协议
- 文件系统、LSM-Tree / B+Tree、WAL
- 性能优化（perf、火焰图、零拷贝、CPU 亲和性）

**与你的背景匹配点：** WebServer → 分布式 KV → 存储引擎，是一条非常自然的路线。你做过的 epoll + 线程池 + mmap + 定时器，全都能复用。LeetCode 650 题 + 算法竞赛基础是加分项。

**岗位分布：** 高度集中在北上深杭，以字节跳动、各大云厂商、数据库创业公司为主。岗位总量不多，但候选人池也小，竞争不像 Java 后端那么卷。

**风险：** 部分基础设施岗需要硕士学历（尤其数据库内核），本科可能需要更强项目背景。

### 方向二：游戏服务器 ⭐⭐⭐⭐

**典型岗位：** 游戏后端（C++/Go）、战斗服务器、匹配系统、AOI 系统

**代表公司：** 腾讯游戏（王者荣耀/和平精英/cf）、网易游戏、米哈游（原神）、莉莉丝、鹰角、叠纸

**你需要的能力：**
- C++ 网络编程 + 多线程 ← 你已经有了
- 游戏服务器框架（帧同步/状态同步、AOI、ECS）
- 熟悉 Linux + 数据库 + Redis
- 了解 protobuf、消息队列

**与你的背景匹配点：** 游戏服务器本质上是一个高实时性、高并发的 C++ 网络服务。你的 WebServer 里 epoll + 线程池 + 定时器是游戏服务器的核心组件。CS144 的 TCP 知识对理解游戏网络协议（KCP/可靠 UDP）也有帮助。

**岗位分布：** 上海最多（米哈游/莉莉丝/鹰角等），深圳（腾讯），广州（网易），杭州。**游戏行业是目前 C++ 招聘需求最大的领域之一**，而且对应届生友好。

**风险：** 游戏行业周期性强，版号政策影响大。加班文化普遍严重（项目上线期 996+）。技术站相对封闭，跳槽跨行业较难。游戏服务器逻辑偏业务，和"底层系统"有差别。

### 方向三：自动驾驶 ⭐⭐⭐⭐

**典型岗位：** 感知系统工程师、规划控制、仿真平台、车载中间件、数据管道

**代表公司：** 小马智行、文远知行、百度 Apollo、蔚来、小鹏、理想、地平线、Momenta

**你需要的能力：**
- C++（现代 C++14/17）+ Python
- 多线程、内存管理、实时系统
- Linux + 中间件（ROS2/DDS/Adaptive AUTOSAR）
- 点云处理、GPU 编程（加分）

**与你的背景匹配点：** 自动驾驶的"中间件/平台组"非常需要 C++ 系统编程能力。你的 WebServer 中学到的 epoll、线程池、零拷贝，在车载域控制器上做数据分发/日志采集时是核心技能。算法题刷得好对于感知/规控岗位的面试也有帮助。

**岗位分布：** 北京（百度/小马/地平线）、上海（蔚来/Momenta）、广州（小鹏/文远）、深圳（比亚迪/元戎启行）。**这个赛道正在快速增长**，很多车企（蔚小理）自研智驾团队在扩张。

**风险：** 行业还在早期，部分公司烧钱速度 > 商业化速度，稳定性不如大厂。部分感知/算法岗位要求硕士，本科更适合做 infra/工具链/平台。

### 方向四：量化交易 / 金融 ⭐⭐⭐

**典型岗位：** 低延迟交易系统、行情数据处理、回测引擎

**代表公司：** 幻方、九坤、明汯、佳期、Citadel、Optiver、Jump Trading

**你需要的能力：**
- C++ 极致性能优化（cache line、branch prediction、lock-free）
- FPGA/网卡 offload（加分）
- Linux 内核旁路（DPDK/XDP）
- 数学/统计学/金融市场基础

**与你的背景匹配点：** 你的算法竞赛背景（1750 分）+ C++ 性能优化兴趣是优势。量化公司非常看重算法竞赛成绩。

**岗位分布：** 上海、深圳、香港。头部量化 C++ 岗位薪资极高（应届 80-150w），但全国加起来可能不到 200 个坑。

**风险：** 门槛极高（头部基本要金牌/硕士），岗位数极少，竞争惨烈。可以先放目标清单的远端。

### 方向六：嵌入式 / IoT ⭐⭐

**典型岗位：** MCU 固件、Linux BSP、RTOS 应用、物联网网关

**代表公司：** 华为、大疆、小米、海康威视、各种车载 Tier1

**为什么不太建议：**

- **你的 epoll/线程池/高并发 知识在这个方向基本用不上**

```
第一志愿：基础架构/分布式存储

  ← 下一个项目 MIT 6.824 分布式 KV 无缝衔接

  ← 需要打造更强的项目背景

  ← 算法竞赛继续刷到 2000+ 分再考虑
  ← 大三如果有机会参加 ICPC/CCPC 区域赛拿牌，这条路会更宽
```

## 实际应对策略

### C++ 后端岗确实少，所以要"广撒网 + 精准备"

- **广撒网**：基础架构、游戏服务器、自动驾驶都投。你的简历（网络编程 + C++ + 算法）在这三个方向的实习生招聘中都是合格的。
- **精准备**：不同方向投不同版本的简历。投基础架构强调分布式 KV/Raft；投游戏强调高并发网络编程；投自动驾驶强调系统编程/中间件。

### 大三投实习的目标公司清单

| 梯队 | 公司 | 方向 |
|------|------|------|
| T0 冲刺 | 字节（基础架构）、腾讯（TEG/云）、阿里云 | 基础架构 |
| T1 主攻 | 米哈游、莉莉丝、鹰角、PingCAP、蔚来、小鹏 | 游戏/数据库/智驾 |
| T2 保底 | 网易游戏、三七互娱、B站、Zoom、Citrix | 游戏/音视频 |
| T3 练手 | 各类中小厂、创业公司（面试练手） | 不限 |

### 如果还是担心，可以做的额外准备

- **Go 作为第二语言**：很多基础架构组是 C++ + Go 混合。字节/腾讯的中间件用 Go 写的很多。花两周学 Go（goroutine + channel），面试时表示"主 C++，Go 也能写"，岗位面扩大一倍。
- **Rust 可以了解但不急**：等你有 1-2 年 C++ 经验后再转 Rust 会很容易，现在学了没项目支撑面试也聊不深。

> 不用担心 C++ 后端岗少 —— C++ 程序员的竞争池本身就比 Java 小得多。你能把 WebServer 做到这个程度 + 650 道算法题 + 一个分布式项目，大三找实习是完全够用的。
