### 下一步行动（按优先级，建议 3-4 周完成）

#### 第 1 步：让服务器"功能完整"（本周）

- [ ] **修复 URL 路径映射** — `parse_request_line` 已解析 `m_path`，但 `do_request()` 忽略它。需要：`m_path` → 拼接 `./root` + 路径 → `stat()` 判断是文件还是目录 → 目录返回 `index.html`
- [ ] **防目录穿越攻击** — 检查 `m_path` 中是否含 `..`，realpath 验证最终路径仍在 `./root/` 下
- [ ] **MIME 类型映射** — 根据文件后缀设置 `Content-Type`（html/css/js/png/jpg/svg/ico/json/pdf），用一个 `std::unordered_map` 即可
- [ ] **修 bench.sh + 跑通首次压测** — 修 CMakeLists.txt 加 `server_single` target 或改 bench.sh 去掉单线程对比；用 wrk 跑出第一组 QPS 数据作为基线

#### 第 2 步：性能优化亮点（下周）

- [ ] **时间轮替换排序链表** — 设计一个 Timing Wheel，槽位数 = (最大超时 / TIME_SLOT)，插入/删除/调整/tick 全部 O(1)。这是面试展示数据结构选型能力的核心亮点
- [ ] **Keep-alive 连接复用验证** — 确保同一连接可以处理多个请求：wrk 默认用 keep-alive，压测时观察连接数变化
- [ ] **POST body 处理** — 如果是 POST 请求且 `Content-Length > 0`，读取 body 并做基本处理（如解析表单数据、JSON）

#### 第 3 步：工程化收尾（第三周）

- [ ] **压测优化循环** — 跑 wrk 在不同并发数(100/500/1000/5000)下的 QPS，找到瓶颈，优化，再测。至少做 2 轮优化迭代

- 对象模型：虚函数表、多重继承内存布局

- 进程/线程模型、clone、信号、内核调度 CFS
- 虚拟内存 → mmap/madvise/mlock、缺页中断

- 网络栈 → TCP 拥塞控制、epoll 在内核中的位置
- 锁/同步机制 → 互斥锁、条件变量、无锁队列

### 方向一：基础架构 / 分布式存储 ⭐⭐⭐⭐⭐

**典型岗位：** 分布式存储引擎、数据库内核、消息队列、RPC 框架、容器运行时

**代表公司：** 字节跳动（基础架构数据库团队——VLDB/SIGMOD 发论文的顶级 infra 团队）、阿里云（PolarDB/Flink/Lindorm）、腾讯云（TDSQL/TBase）、PingCAP（TiDB）、OceanBase、华为（高斯 DB）

**你需要的能力：**
- 网络编程 / 多线程 / 锁 / 无锁数据结构 ← 你正在做
- Raft/Paxos 等分布式一致性协议
- 文件系统、LSM-Tree / B+Tree、WAL
- 性能优化（perf、火焰图、零拷贝、CPU 亲和性）

**与你的背景匹配点：** WebServer → 分布式 KV → 存储引擎，是一条非常自然的路线。你做过的 epoll + 线程池 + mmap + 定时器，全都能复用。LeetCode 650 题 + 算法竞赛基础是加分项。

**岗位分布：** 高度集中在北上深杭，以字节跳动、各大云厂商、数据库创业公司为主。岗位总量不多，但候选人池也小，竞争不像 Java 后端那么卷。

**风险：** 部分基础设施岗需要硕士学历（尤其数据库内核），本科可能需要更强项目背景。

**⚠️ 实习 JD 校准（2026.05）：**

- **字节数据库内核 JD**：强 AI 融合趋势（AI 原生数据库、向量搜索、LLM + NL2SQL），偏好硕士/顶会论文，本科较难直接命中。策略：主攻基本功（B+树/LSM-Tree/WAL/Raft），AI 建立认知即可。
- **字节基础设施存储 JD** ⭐ **更适合你**：职位要求仅为"算法+数据结构+一门语言"，你的 WebServer（epoll+mmap+线程池+零拷贝）与对象/文件存储的数据面直接对应。MIT 6.824 Raft 对应存储元数据管理。无 AI 技能硬性要求——团队做的是给大模型提供存储底座（支撑豆包/方舟），不是做 AI 算法。本科友好度高得多。
- **结论**：同是字节基础架构，存储方向比数据库内核更适合你当前背景。投递时优先存储，数据库内核作为冲刺。

**代表公司：** 华为、大疆、小米、海康威视、各种车载 Tier1

**为什么不太建议：** 
- 需要硬件知识（SPI/I2C/CAN、示波器、数据手册），你的背景要从头补
- 技术栈偏 C（非 C++），很多是裸机或 RTOS，没有操作系统抽象
- 和互联网行业的薪资差距明显
- **你的 epoll/线程池/高并发 知识在这个方向基本用不上**

---

## 基于你背景的推荐排序

```
第一志愿：基础架构/分布式存储
  ← 你目前的积累（epoll、线程池、mmap、零拷贝、CS144）全部用得上
  ← 下一个项目 MIT 6.824 分布式 KV 无缝衔接
  ← 技术壁垒高，不可替代性强
  ← 需要打造更强的项目背景

第二志愿：游戏服务器
  ← 也是 C++ 网络编程，WebServer 经验直接用
  ← 岗位多，对应届生友好，尤其是上海
  ← 缺点：技术栈较封闭，加班严重
  ← 如果基础架构岗面试不顺，这是非常好的保底方向

第三志愿：自动驾驶平台
  ← C++ 系统编程 + 中间件方向，和你的底子匹配
  ← 行业在上升期，招人多
  ← 需要额外学一些 ROS2/车载中间件

备选：量化 C++
  ← 算法竞赛继续刷到 2000+ 分再考虑
  ← 大三如果有机会参加 ICPC/CCPC 区域赛拿牌，这条路会更宽
```

---

## 实际应对策略

### C++ 后端岗确实少，所以要"广撒网 + 精准备"

- **广撒网**：基础架构、游戏服务器、自动驾驶都投。你的简历（网络编程 + C++ + 算法）在这三个方向的实习生招聘中都是合格的。
- **精准备**：不同方向投不同版本的简历。投基础架构强调分布式 KV/Raft + 数据库内核知识；投游戏强调高并发网络编程；投自动驾驶强调系统编程/中间件。
- **关于 AI 化 JD**：当前数据库内核岗位越来越多带上"AI 原生/向量搜索/LLM 集成"等关键词。本科生策略——主攻数据库内核基本功（B+树/LSM-Tree/WAL/Raft），这些是硬通货。AI 部分通过技术博客和论文建立认知即可，面试时能聊"为什么向量数据库在 RAG 场景下火"就足够，不需要做一个 AI 项目来证明。

### 大三投实习的目标公司清单

| 梯队 | 公司 | 方向 |
|------|------|------|
| T0 冲刺 | 字节（基础架构）、腾讯（TEG/云）、阿里云 | 基础架构 |
| T1 主攻 | 米哈游、莉莉丝、鹰角、PingCAP、蔚来、小鹏 | 游戏/数据库/智驾 |
| T2 保底 | 网易游戏、三七互娱、B站、Zoom、Citrix | 游戏/音视频 |
| T3 练手 | 各类中小厂、创业公司（面试练手） | 不限 |

### 如果还是担心，可以做的额外准备

- **Go 作为第二语言**：很多基础架构组是 C++ + Go 混合。字节/腾讯的中间件用 Go 写的很多。花两周学 Go（goroutine + channel），面试时表示"主 C++，Go 也能写"，岗位面扩大一倍。
- **Rust 可以了解但不急**：等你有 1-2 年 C++ 经验后再转 Rust 会很容易，现在学了没项目支撑面试也聊不深。

> 不用担心 C++ 后端岗少 —— C++ 程序员的竞争池本身就比 Java 小得多。你能把 WebServer 做到这个程度 + 650 道算法题 + 一个分布式项目，大三找实习是完全够用的。
